STORE_LISTING.md を最新の変更内容に基づいて更新してください。

## 手順

1. **変更差分の調査**: 引数で指定されたコミットハッシュ（またはバージョンタグ）から HEAD までの変更を調査する
   - `git log <commit>..HEAD --no-merges --oneline` でコミット一覧を確認
   - `git log <commit>..HEAD --no-merges --format="%h %s"` で各コミットの概要を確認
   - 引数が未指定の場合は、STORE_LISTING.md の冒頭に記載されているバージョン番号を元に、そのバージョンに対応するコミットを特定する

2. **現在のバージョン確認**: `public/manifest.json` の `version` フィールドを確認する

3. **変更内容の分類**: 各コミットを以下のカテゴリに分類する
   - 新機能（feat）
   - バグ修正（fix）
   - UI/UX改善
   - パフォーマンス改善
   - その他

4. **STORE_LISTING.md の更新**: 以下のセクションを更新する

   **a. 冒頭のアップデート情報セクション（📢）**
   - 現在のバージョン番号に更新
   - 新機能・改善点をユーザー向けに分かりやすく箇条書き
   - 旧バージョンの情報は1世代前まで残す（それ以前は統合して簡潔に）

   **b. 概要セクション（📖 概要）**
   - 新しいエンジンや主要機能が追加された場合に更新

   **c. 機能紹介セクション（✨ 機能紹介）**
   - 新機能を適切なカテゴリに追加
   - 既存の説明文が古い場合は更新

   **d. その他のセクション**
   - 使い方（🔧）、APIキー（🔑）、サービス（🙏）は、大きな変更がある場合のみ更新

5. **文体ルール**:
   - ユーザー向けのカジュアルな日本語で書く（ストア掲載文のため）
   - 技術的な内部実装の詳細は含めない
   - 新機能は「！」で強調し、ユーザーメリットを明示する
   - 絵文字は控えめに使用（各セクション見出しのみ基本OK）

6. **最終確認**: 更新後のファイルを読み返し、全新機能が反映されていることを確認する
